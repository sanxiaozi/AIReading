#### 根本原因（代码层面已确认）

`src/app/sitemap.ts` 中 `buildUrl()` 函数的逻辑：
- `locale === 'zh'` → 直接返回无前缀路径（如 `/about`, `/category/business`）
- `locale === 'en'` → 返回 `/en/about`, `/en/category/business`

但实际路由结构中，**中文版页面部署在 `/zh/` 前缀下**，无前缀路径全部返回 404。

```typescript
// 当前错误逻辑（sitemap.ts line 37-38）
if (locale === 'zh') {
  return `${baseUrl}${path}`;  // ❌ 生成 /about → 404
}
// 正确应为：
return `${baseUrl}/zh${path}`;  // ✅ 生成 /zh/about → 200
```

```

```

