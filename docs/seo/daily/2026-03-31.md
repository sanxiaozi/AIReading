### ❌ 问题 1：Sitemap 中大量 URL 返回 404

**严重程度**: 🔴 CRITICAL  
**发现时间**: 2026-03-31

Sitemap.xml 中包含的 `/about`、`/faq`、`/category/*`（无语言前缀）等路径全部返回 404，实际上这些页面存在于 `/zh/` 和 `/en/` 路径下。

**受影响 URL 示例**：
```
https://aireading.com/about        → 404 ❌  (实际地址: /zh/about 或 /en/about)
https://aireading.com/faq          → 404 ❌
https://aireading.com/category/business → 404 ❌
https://aireading.com/category/psychology → 404 ❌
... (共约 20+ 无语言前缀 URL)
```

**正常 URL**：
```
https://aireading.com/zh/about      → 200 ✅
https://aireading.com/en/about      → 200 ✅
https://aireading.com/zh/category/business → 200 ✅
```

**影响**：Google/Bing 爬虫将尝试爬取这些 404 URL，导致：
- 索引预算浪费
- 搜索引擎降低对网站的信任评分
- 这些页面永远不会被收录

**修复方案**：
1. 更新 `sitemap.xml` 生成逻辑，所有 URL 需包含语言前缀 `/zh/` 或 `/en/`
2. 或在 `next.config.ts` 中添加重定向规则，将无前缀 URL 重定向到默认语言版本

### ⚠️ 问题 2：书籍页面路径 308 重定向后 404

**严重程度**: 🔴 CRITICAL  
**发现时间**: 2026-03-31

`/zh/book/1/`（带尾斜杠）→ 308 重定向到 `/zh/book/1`（无尾斜杠）→ 404

```
https://aireading.com/zh/book/1/  → 308 → /zh/book/1 → 404 ❌
https://aireading.com/en/book/1/  → 308 → /en/book/1 → 404 ❌
```

**实际工作路径**：
```
https://aireading.com/books/1     → 200 ✅  (无语言前缀)
```

**分析**：sitemap 使用的是 `/books/[id]` 格式（无语言前缀），而 robots.txt 允许的是 `/zh/book/` 和 `/en/book/`。路由配置存在不一致。

**修复方案**：确认实际路由结构，统一 sitemap、robots.txt 和实际路径。
