## 3️⃣ Podcastfy 技术选型 Thread（5条）

### Tech Thread 1/5
```
🎙️ 为什么我们选择 Podcastfy 而不是传统 TTS？

一个简单的对比：

Traditional TTS: 
"原子习惯这本书讲述了..."（机械、单调）

Podcastfy:
A: "你听说过原子习惯吗？"
B: "当然！我觉得最精彩的是..."

哪个你更想听？🧵👇
```

### Tech Thread 2/5
```
Podcastfy = NotebookLM 的开源版

✅ 双人对话形式
✅ 自然的语气和停顿
✅ 互动式讨论而非朗读
✅ 基于 GPT + ElevenLabs

最重要的：它让"听书"变得有趣

参考 @NotebookLM 的 Audio Overview 功能
```

### Tech Thread 3/5
```
技术实现细节（for developers 🤓）：

1. 输入：书籍摘要（markdown）
2. GPT-4 生成对话脚本（2 个角色）
3. ElevenLabs TTS 分角色朗读
4. 自动合成音频 + 添加背景音乐
5. 输出：播客风格的音频

GitHub: podcastfy
开源工具 FTW 🚀
```

### Tech Thread 4/5
```
为什么开源工具很重要？

✅ 透明度：知道它怎么工作
✅ 可控性：可以修改和优化
✅ 社区：有问题可以找人讨论
✅ 成本：不依赖昂贵的 API

Build in Public 的精神：
站在巨人的肩膀上创新
```

### Tech Thread 5/5
```
结果：

生成一集 30 分钟播客需要：
⏱️ ~15 分钟生成时间
💰 ~$2-3 成本（API 调用）
🎧 质量堪比专业播客

trade-off: 速度换质量
但用户体验值得这个投入

下周开始生成第一本书 📚
```
